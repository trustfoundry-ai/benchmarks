# Security Policy

## Reporting a Vulnerability

Please do not open public issues for suspected vulnerabilities or exposed secrets. Report them privately to a repository owner with:

- A short description of the issue.
- Steps to reproduce or validate it.
- Any affected files, workflows, or benchmark artifacts.

We will acknowledge credible reports and coordinate remediation before public disclosure.

## Public Benchmark Data

This repository is public. Do not include private customer data, production secrets, private API responses, or non-public evaluation datasets in pull requests.
